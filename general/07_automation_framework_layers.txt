- Automation Framework Layers

1) Framework Layer (defines the overall flow and structure of the automation process)
2) Automation API Layer (interacts directly with the application under test)
3) Test Scripts Layer (automated test cases and test scripts)
4) Utility and Helper Layer (functions for reading configuration files, handling file operations)
5) Test Library Layer (reusable modules, components, packages)
6) Configuration Layer (URLs, credentials, and environment-specific details)
7) Logging and Reporting Layer (logging information during test execution and generating detailed reports)
8) Test Data Layer (creation, storage, and retrieval of test data)
